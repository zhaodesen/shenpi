# CRM 签单审批演示系统

一个覆盖权限配置、人员管理、签单申请、多级审批和客户库的完整 CRM 前端流程演示。

[English](./README.md) | [简体中文](./README.zh-CN.md)

## 主要功能

- 组织架构、组、角色与权限配置
- 带功能级权限控制的人员管理
- 动态产品申请表单
- 为新客户或已有客户发起签单申请
- 财务首审与后续多级审核
- 审核阶段之间的申请人信息补充流程
- 面向提交人、审核人、上级和管理员的客户可见性规则
- 内置演示身份和可重置的本地数据

## 技术栈

Vue 3 · Vite 6 · Vue Router · Pinia · Element Plus · localStorage

## 快速开始

```bash
npm install
npm run dev
```

如果 Vite 未输出其他地址，默认访问 `http://localhost:5173`。

## 审批流程

```text
提交签单申请
  → 财务审核
  → 申请人补充必要信息
  → 后续审核人逐级审批
  → 客户与产品进入客户库
```

进入下一审核人前，必须同时满足“财务已同意”和“申请人已完成补充”两个条件。

## 主要路由

| 路由 | 模块 |
| --- | --- |
| `/org` | 组织架构与权限配置 |
| `/personnel` | 人员管理 |
| `/product-form` | 产品申请表单 |
| `/apply-create` | 新建签单申请 |
| `/apply-list` | 申请列表与补充任务 |
| `/review` | 审批工作台 |
| `/customer` | 客户库 |

## 演示数据

通过右上角身份切换器，可分别体验销售、财务、审核人、上级和管理员视角。数据保存在浏览器中，可通过“重置演示数据”恢复初始状态。

## 项目结构

```text
src/
├── components/   # 动态表单与公共组件
├── constants/    # 权限点与状态
├── layouts/      # 主布局
├── mock/         # 演示种子数据
├── router/       # 路由与权限守卫
├── stores/       # Pinia 业务 Store
└── views/        # 功能页面
```

## 项目边界

本项目是前端流程演示，使用本地模拟数据，不包含可直接用于生产环境的 CRM 后端。
